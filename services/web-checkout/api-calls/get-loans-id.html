GET /loans/:id New

Consumes: loan-api

Screen

PageLoan Detail (/dashboard/loans/:id)
ComponentLoanDetailPage → LoanSummaryCard, LoanTermsPanel, PaymentSchedule
TriggerPage load — called when navigating from the loan list or via direct URL

Description

Fetches the full detail view for a single loan. This is the primary data source for the loan detail page, which renders the loan summary card, terms breakdown, repayment schedule, and the payment history tab.

Frontend Logic

  • Loading state: Renders a full-page skeleton with placeholder cards for summary, terms, and schedule sections
  • Optimistic navigation: If the loan was already fetched in the list view, the cached LoanDetailDto from loanStore is shown immediately while a background revalidation runs
  • Status-dependent UI: The page conditionally renders an "Edit Loan" button (status pending), a "Make Payment" button (status active), or a completion badge (status completed)
  • Polling: When loan status is pending, the page polls this endpoint every 10 seconds to detect approval/rejection updates
  • Error handling: On 404 (loan not found), redirects to the dashboard with a "Loan not found" toast; on 403, shows an access-denied page

Request

ParameterLocationDescription
idPathThe loan identifier, extracted from the route param

Response

Returns a LoanDetailDto with full loan details including status, terms, amount, purpose, and creation date. The frontend maps status to a color-coded badge via getLoanStatusConfig().
