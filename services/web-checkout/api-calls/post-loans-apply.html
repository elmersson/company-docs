Description

Submits a new loan application from the checkout flow. The application form collects loan amount, purpose, and desired term across a multi-step wizard. On successful submission the user is redirected to a confirmation screen with their application reference number.

Frontend Logic

  • Form validation: Client-side validation via react-hook-form with Zod schema — amount must be between 1,000 and 500,000, term between 6 and 60 months
  • Optimistic UI: The submit button enters a loading state with spinner immediately on click; disabled to prevent double-submission
  • Success flow: On 201 response, navigates to /apply/confirmation with the new loan ID, clears form draft from sessionStorage
  • Error handling: On 422 (validation error), maps server field errors back to the form fields; on 429 (rate limit), shows a “Please wait” toast with countdown; on 5xx, shows a retry dialog
  • Draft persistence: Form state is auto-saved to sessionStorage every 2 seconds so the user can resume if they navigate away

Request

Sends a LoanApplicationDto as JSON request body. Requires Authorization: Bearer <token> header.

FieldTypeDescription
amountnumberRequested loan amount (1,000–500,000)
purposestringPurpose of the loan (free text)
termnumberLoan term in months (6–60)

Response

Returns a LoanResultDto with the application outcome. The frontend checks the status field to determine whether to show the success confirmation or a “pending review” interstitial.
