GET /loans New

Consumes: loan-api

Screen

PageLoan Dashboard (/dashboard/loans)
ComponentLoanListPage → LoanTable
TriggerPage load — called on mount and on pull-to-refresh

Description

Fetches the full list of loans for the authenticated user. The response populates the main loan dashboard table where users can see all their active, pending, and completed loans.

Frontend Logic

  • Loading state: Shows a skeleton table with 5 placeholder rows while the request is in flight
  • Empty state: If the response array is empty, renders a NoLoansIllustration with a CTA to apply for a new loan
  • Sorting: Client-side sorting by createdAt (newest first) applied after fetch; user can toggle column sort
  • Filtering: Client-side status filter tabs (All / Active / Completed / Rejected) computed from the response
  • Caching: Response is cached in the loanStore (Zustand) and revalidated on window focus via useSWR
  • Error handling: On 401, redirects to login; on 5xx, shows a toast notification with retry button

Request

No request body. Requires Authorization: Bearer <token> header (injected by the API client interceptor).

Response

Returns an array of LoanDetailDto objects. The frontend maps these to LoanRow view models for table rendering.
