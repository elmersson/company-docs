POST /loans/:loanId/payments New

Consumes: loan-api

Screen

PageLoan Detail — Payments Tab (/dashboard/loans/:loanId)
ComponentLoanDetailPage → PaymentsTab → MakePaymentModal
TriggerUser click — fires when the user fills in the payment amount and clicks “Submit Payment” in the modal

Description

Submits a payment for a specific loan. The “Make Payment” button is only visible when the loan status is active. Opens a modal where the user enters the payment amount, which defaults to the next scheduled installment amount.

Frontend Logic

  • Default amount: Pre-fills the payment amount field with the next scheduled installment from the loan’s repayment schedule
  • Validation: Amount must be positive and cannot exceed the remaining loan balance; validated client-side before submission
  • Confirmation step: After filling in the amount, a confirmation dialog shows the payment summary before the actual API call fires
  • Success flow: On 201, closes the modal, shows a “Payment submitted” success toast, and refetches the payment history list
  • Error handling: On 400 (invalid amount), shows inline form error; on 409 (payment already processing), shows an alert to wait; on 5xx, shows a retry dialog with the payment details preserved
  • Optimistic list update: The new payment is prepended to the payment history list with a “Processing” badge while the background refetch confirms it

Request

Sends a CreatePaymentDto as JSON request body. The loanId is extracted from the route parameter.

ParameterLocationDescription
loanIdPathThe loan identifier
amountBodyPayment amount

Response

Returns a PaymentDto confirming the payment was accepted. Includes the payment id, status (processing | completed), and createdAt timestamp.
