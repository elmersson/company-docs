GET /loans/:loanId/payments New

Consumes: loan-api

Screen

PageLoan Detail — Payments Tab (/dashboard/loans/:loanId)
ComponentLoanDetailPage → PaymentsTab → PaymentHistoryTable
TriggerTab switch — called when the user clicks the “Payments” tab on the loan detail page, and on initial load if the tab is the default

Description

Fetches the payment history for a specific loan. Displayed as a chronological table in the Payments tab of the loan detail page. Each row shows the payment date, amount, status, and a reference number.

Frontend Logic

  • Lazy loading: Payment history is only fetched when the Payments tab is activated, not on initial page load — reduces unnecessary API calls for users who only view loan details
  • Loading state: Shows a skeleton table with 3 placeholder rows while the request is in flight
  • Empty state: If no payments exist, renders a “No payments yet” message with the next scheduled payment date and a “Make Payment” CTA button
  • Sorting: Server returns payments newest-first; the frontend allows toggling sort by date or amount via column headers
  • Status badges: Each payment row shows a color-coded status badge — green for completed, yellow for processing, red for failed
  • Error handling: On failure, shows an inline error banner within the tab (does not affect the rest of the loan detail page) with a retry button
  • Auto-refresh: When any payment has processing status, the list polls every 5 seconds until all payments settle

Request

No request body. The loanId is extracted from the route parameter. Requires Authorization: Bearer <token> header.

ParameterLocationDescription
loanIdPathThe loan identifier

Response

Returns an array of PaymentSummaryDto objects. Each object includes id, amount, status, createdAt, and referenceNumber. The frontend maps status to badge colors via getPaymentStatusConfig().
