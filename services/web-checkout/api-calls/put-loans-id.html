TriggerUser click — fires when the user modifies loan terms in the edit modal and clicks “Save Changes”

Description

Updates an existing loan application. Only available when the loan status is pending — the “Edit Loan” button is conditionally rendered based on status. Used to modify the loan amount, purpose, or term before final approval by the lender.

Frontend Logic

  • Pre-fill: The edit form is pre-populated from the cached LoanDetailDto in the Zustand store; no additional fetch required
  • Optimistic update: On submit, the local store is updated immediately with the new values; if the request fails, the store rolls back to the previous state
  • Success flow: On 200, closes the edit modal, shows a “Loan updated” toast, and triggers a background revalidation of the loan detail
  • Error handling: On 409 (conflict — loan already approved), shows an alert explaining the loan can no longer be edited and refreshes the page; on 422, maps field errors back to the form
  • Guard: If the user navigates to the edit route for a non-pending loan, redirects back to the detail page with a toast

Request

Sends an UpdateLoanDto as JSON request body. The loan id is extracted from the route parameter.

ParameterLocationDescription
idPathThe loan identifier
amountBodyUpdated loan amount (optional)
purposeBodyUpdated purpose (optional)
termBodyUpdated term in months (optional)

Response

Returns the updated LoanDetailDto. The frontend replaces the cached loan in the store and re-renders the detail page with the new values.
